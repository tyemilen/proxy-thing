## Конфиг

По умолчанию берёт config.json из текущей директории, можно указать свой:

```
proxy-thing /cool/path/to/conf.json
```

Пример конфига

```json
{
	"address": "127.0.0.1:1337",
	"limits": {
		"banTime": 60,
		"gcIntervalTime": 60,
		"pingIntervalTime": 21,
		"maxTimeoutRetries": 5,
		"maxResponseRetries": 5,
		"maxRedirects": 3
	},
	"paths": {
		"crt": "proxy-thing.crt",
		"key": "proxy-thing.key",
		"proxies": "proxies.txt"
	}
}
```

### Параметры

#### Основные
| Параметр  |                                                                   |
| --------- | ----------------------------------------------------------------- |
| `address` | Адрес и порт, где поднимается сервер                              |

#### limits
| Параметр             |                                                         |
| -------------------- | ------------------------------------------------------- |
| `banTime`            | Сколько секунд хост считается плохим и не используется  |
| `gcIntervalTime`     | Как часто проверять, пора ли разбанить хосты            |
| `maxTimeoutRetries`  | Сколько раз повторять запрос при таймауте               |
| `maxResponseRetries` | Сколько раз повторять запрос при плохом ответе (403)    |
| `maxRedirects`       | Максимальное количество редиректов                      |

#### paths
| Параметр  |                       |
| --------- | --------------------- |
| `crt`     | Путь к сертификату    |
| `key`     | Путь к ключу          |
| `proxies` | Путь к списку прокси  |

## Формат прокси
Обычный список по одному прокси на строку

```
vless://...
ss://...
```


## Генерация ключа и сертификата
```
openssl genrsa -out proxy-thing.key 2048
openssl req -new -x509 -days 3650 -key proxy-thing.key -out proxy-thing.crt -subj "/CN=Proxy-Thing"
```

Сертификат нужно установить на клиенте иначе соединение будет считаться недоверенным
